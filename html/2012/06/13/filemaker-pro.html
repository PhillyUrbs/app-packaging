FileMaker Pro v8.x will not install silently with a properly created MST (although it will install using MSIEXEC /I <path to msi> TRANSFORMS=<path to mst>).  In order for the silent install to work, you need to modify the Assisted Install.txt to contain your group's license info.

Assisited Install.txt Sample

Name= .

Organization= Company Name

License Key= xxxxx-xxxxx-xxxxx-xxxxx-xxxxx-xxxxx-xxxxx

... rest of the stuff can be left as per their defaults ...

The organization name must be entered as it appears under "Licensee" on your license agreement.  The company name is  all upper case as it appears on the license agreement. One may change organization case from "COMPANY NAME" to something like "Company Name".

License Key works using it's dashed form.

You can use /QB or /Passive on the command line as per a regular MSI install to make it run in an automated mode.

Once the main File Maker Pro is installed off the CD or install point, you may then run your update commands to bring File Maker from Ver 1 up to Ver 3.  The updates must be run after the main install.

With 8.5, the updater to v2 is a full featured installer.
There is no need to install the original version first.
The only thing, one needs to change, is the following property: LanguagePackInstaller=0.
You have to do it with an MST or edit the MSI directly.
Applying it from the command line won't work, because this is not a public property.

Shortcuts are best managed through the 'Assisted Install.txt' file. See this for the available options in this file.

Main:

  msiexec.exe /package "\\SERVER\INST\File_Maker PRO\FileMaker Pro 8.msi" /passive



V2Upd:

msiexec.exe /package "\\SERVER\INST\FileMaker Pro 8.0v2 Updater\Files\FileMaker Pro 8.msi" /passive



V2 & V3 Combined:

msiexec.exe /package "\\SERVER\INST\FileMaker Pro 8.0v2 Updater\Files\FileMaker Pro 8.msi" /passive /update "\\SERVER\INST\FileMaker Pro 8.0v3 Updater\Files\FileMakerPro80v3.msp"



So for a fresh install, you could simply run the Main: followed by the V2 & V3 Combined line.  If you want to allow the user to cancel the install, you can /QB instead of /Passive.
