This is for the divace replacement used as language lab software used in language courses. It was previously known as divace.

For the media assistant duo each installation needs to have unique client identification.

Do a standard installation then import the following registry settings to setup the program for use as a unique client connected to the server.

Windows Registry Editor Version 5.00

[HKEY_LOCAL_MACHINE\SOFTWARE\Sanako]

[HKEY_LOCAL_MACHINE\SOFTWARE\Sanako\Shared Components]

[HKEY_LOCAL_MACHINE\SOFTWARE\Sanako\Shared Components\Common]
"ToLabNumber"=dword:00000001

[HKEY_LOCAL_MACHINE\SOFTWARE\Sanako\Shared Components\NetCommPlatform]

[HKEY_LOCAL_MACHINE\SOFTWARE\Sanako\Shared Components\NetCommPlatform\Client]
"ClientId"=dword:00000001

The above settings are for client number 1, change the numbering for each unit, the number must be saved as hexidecimal.

For novel users put your own variant of the following in the application object - :

#regedit.exe /s "I:\Language\Sanako Student\Client\%computername%\client.reg"

to be placed in run after termination. It just imports the reg file containing the client settings for each computer. The identification being the computername, and each reg file variant being saved in a folder named for the computers hostname. Alternatively just save each reg file as hostname.reg and import %computername%.reg

When users log out of sanako duo the process is terminated and it no longer works for the next user until someone starts the program.

Create a shortcut to the following (assuming installation to the default location)

%*PROGRAMFILES%\Sanako\Shared Components\NetCommPlatform\RCLNT.EXE

and place it in %*COMMONSTARTUP% so that whenever anyone logs on the client is started automatically.

Basically, our client tries to be the very last thing to shutdown.  The reason for this is that students could start the shutdown process and cancel it - thus leaving our client disabled and no teacher control possible.  However, if you add the following DWORD to each student workstation then our service no longer tries to be the last and just joins the cue as per any other process.

Create a new DWORD value

FastShutdown = 1

to the following registry branch

HKEY_LOCAL_MACHINE\SOFTWARE\Sanako\Shared Components\NetCommPlatform\Client
