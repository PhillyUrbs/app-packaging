Basically, this is an ActiveX IE Plugin which installs the first time you enter the website.  If your users are in a locked down environment, the installation will fail.  In order to create this package, we used WPS and did a capture before and after.

The actual ActiveX plugin is 7 DLLs and 1 .INF.  Using WPS, setup to copy the 7 DLLs to C:\WINNT\Downloaded Program Files and register the DLLs.  We used WPS to register the DLLs in the order listed in the .INF and have had no problems so far.

If you would like the ActiveX plugin to display under Downloaded Program Files, you will also need to add some regkeys as listed below.

Windows Registry Editor Version 5.00

[HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Code Store Database\Distribution Units\{45874228-A445-40DC-962B-EC15559B1741}]
"SystemComponent"=dword:00000000
"Installer"="MSICD"

[HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Code Store Database\Distribution Units\{45874228-A445-40DC-962B-EC15559B1741}\Contains]

[HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Code Store Database\Distribution Units\{45874228-A445-40DC-962B-EC15559B1741}\Contains\Files]
"C:\\WINDOWS\\Downloaded Program Files\\ieop_vcm_77a0cd7e-b03c-435b-bf33-3f207188a6ea.dll"=""
"C:\\WINDOWS\\Downloaded Program Files\\ieop_key_77a0cd7e-b03c-435b-bf33-3f207188a6ea.dll"=""

I did the following:

1. Started Snapshot
2. Logged in to Siebel Server and installed active X control
3. Finished Snapshot
4. Customized Snapshot and tested it. When installed under administrator privileges it was fine, but if another user logged in, instead of "Siebel Option Pack" in "Downloaded Program Files" only a key {6d18ad12-bde3-4393-b311-099c346e6df9} was visible.
5. I added a property to the MSI package: ALLUSERS = 1. After that the active X was also registered to all users when installed with administrative privileges
