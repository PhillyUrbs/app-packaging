This package uses the same kind of answer file as Reader and you can either use the tool Adobe provides or edit the Abcpy.ini file your self. Here are an example of one I used for deployment:

[OEM Install]

SetupTitle= (put the title for your installation)

SetupCaptionBarTitle= (put the caption bar title)

DisplayWelcomeDlg=NO

DisplayEULA=NO

DisplayTypeOfInstallDlg=NO

DisplaySelectDestDirDlg=NO

DisplayCustomDlg=NO

DisplayUserInfoDlg=NO

DisplayConfirmRegDlg=NO

DisplayStartCopyDlg=NO

DisplayFinishDlg=NO

DisplayFinalMessage=NO

DisplayRebootDlg=NO

DisplayMaintUIDlg=NO

DisplayBillboards=NO

ProgGroupName=

DefaultDestDir=

UserName= ( Name of division or company)

UserCompanyName= (put your company name here)

UserSerialNumber= (put the product serial # here)

DisplayBackGround=YES

DisplayProgressBar=NO

ProgressBarStart=NO

ProgressBarEnd=NO

IgnoreInstallWarnings=YES

CloseRunningApps=NO

InstallModeSilent=YES

UninstallCombineVersions=YES

UninstallPreviousVersions=YES

UninstallModeSilent=YES

PDFOutputFolder=

If you installing the Acrobat 5 above the Acrobat Reader 7.x.x you will find, that the pdf files from a URL does'nt work any longer to open up in IE.


To fix this you have to change the registry key:


HKEY_CLASSES_ROOT\SOFTWARE\Adobe\Acrobat\Exe

Default = ProgramPathToReader (AcroRd32.exe)


Then the pdf from the internet will open up in IE again.

In order for Acrobat 5.x to run correctly for unprivileged users, you must allow the general users (e.g. the BUILTIN\Users group) change/write privileges to the following directories:


These Registry keys need their permissions opened up:

      HKCR\AFormAut.App

      HKCR\AFormAut.App.1

      HKCR\CLSID\{7CD069A1-50AA-11D1-B8F0-00A0C9259304}

      HKCR\TypeLib\{7CD06992-50AA-11D1-B8F0-00A0C9259304}\1.0

      HKCR\TypeLib\{78165D71-DF28-11D3-9A89-005004A56D53}\1.0

      HKLM\SOFTWARE\Adobe\Acrobat Distiller\5.0\PrinterJobControl


These directories need their permissions opened:

      \Program Files\Adobe\Acrobat 5.0\Distillr\Settings

      \Program Files\Adobe\Acrobat 5.0\Distillr\Tmp

      \Program Files\Common Files\Adobe\Color

      \Program Files\Common Files\Adobe\TypeSpt

      \Program Files\Common Files\Adobe\Web

      \Windows\System32\spool\drivers\w32x86\2


And, by the way, the program should generate the following files itself upon opening. Make sure you change the permissions on their folders (or the files themselves) before you run Acrobat as a local administrator, otherwise the general users won't be able to overwrite them as needed:

      \Program Files\Common Files\Adobe\ColorACECache.lst

      \Program Files\Common Files\Adobe\TypeSptAdobeFnt.lst
