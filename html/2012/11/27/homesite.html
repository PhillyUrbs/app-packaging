When repackaging:


1) Capture the reboot

2) Run the application as admin before taking second snapshot


Detail: The setup requests a reboot and places actions in the RUNONCE area of the registry that require administrative rights to the system. Other problems surfaced when the application was run for the first time as a normal user (not an admin.) If the first launch was by an admin everything was fine- although nothing outside HKCU and the USERDATA directory appears to be changed on first launch, play it safe and run the application once as an administrator before taking your second snapshot in the repackaging process.

To package this app for a normal user, try to give Full to registry permissions to these keys:


HKCR\TopStyleLite.Application

HKCR\CLSID\{9A0EB8C5-3ADB-11D3-BCEE-00C04F72DA55}

HKCR\TypeLib\{F7EE4E9B-CD64-11D1-8C81-0060973D093D}

HKCR\TypeLib\{5C21BCD5-F27A-11D2-BB98-00C04F72DA55}


Also apply to "HKCR\Interface" if you still have problems launching the product as a non-administrator user.
