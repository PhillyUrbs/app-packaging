To disable Auto Sing In you need:

1. To place/edit file shared.xml in location %Appdata%\Skype

2. Variable SaveCredential must be equal to 0 insile the file.

Example:


<?xml version="1.0"?>

<config version="1.0" serial="21" timestamp="1318926085.2">

    <UI>

        <Login>

            <SaveCredentials>0</SaveCredentials>

        </Login>

    </UI>

</config>

As others have said, you can get a MSI installer for Skype from their website:
http://www.skype.com/intl/en-us/business/download/

After downloading that, there were only a few customizations that I made in my package.


The Skype installer by default puts some registry entries in HKCU, so every user that runs it for the first time after it installs will get the MSI self-repair window so that those can be created.


To fix that, I simply dropped the Feature -> SkypeRegistryCurrentUser row from the FeatureComponents table using Orca, and those registry entries were never installed. Skype.exe will create ones it needs when run. As an added bonus, this also stops Skype from running upon login, because one of those installed HKCU registry entries is what makes that happen.


I also dropped the row for the desktop shortcut from the Shortcuts table in Orca. Done!
