118
Command Line

You will need to copy the first three CDs into directories naming them disk1 through disk3 in order for the silent install to recognize them.


For a standard installation use:


disk1\setup\setup.exe -silent


For a responsefile installation use


disk1\setup\setup.exe -silent -options "path\responsefile.txt"


A sample responsefile can be found at disk1\util

A detailed desciption of the available options can be found in disk1\readme\yourlanguage\install.html


For removal use:


%ProgramFiles%\IBM\Rational\SDP\6.0\rad_prod\_uninst\uninstall.exe -silent

Note

The setup running a silent (un)installation will fork into java and quit right afterwards, just like all other silent installers in the Websphere series. You will need to watch the java.exe task until it ends. (i.e. using the taskwait.exe tool)

118
Note

Program updates and interrim fixes can be automated with the rpu utility. Place a 'policy.xml' file in <installdir>\updater\eclipse\configuration, a sample policy.xml file is (normaly) supplied with  the updates.

run <installdir>\updater\eclipse\launch.bat -auto


You need to update the updater(rpu) to 6004 for this to work. The updater update needs to be run twice to work correctly.


You will need to delete the policy.xml file after running the updates. If it points to a inaccessable location it have influense on the eclipse startup time.


My deveoppers have asked for a shorter installation path than the default path.
