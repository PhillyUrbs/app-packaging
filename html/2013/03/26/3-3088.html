Obtaining the files:


The original files for the desktop manager software came in a self-extract exe file. This exe will extract a lot of file. You can extract the files with winzip. There is an msi for the desktop manager but you can not run the msi without Running their exe. This package was made with Installshield.


Installation of the desktop manager :


First you need to find where the files were extracted. Then you need to run this command line to make a response file (setup.iss) for installshield : setup.exe /r /f1C:\Setup\setup.iss (whereever you want!)

Then you install the software with all the options you need.


Then copy the setup.iss file in the same directory of the MSI. Put the installation to be per-machine instead of per-user, without reboot.


Instalation command line :

setup.exe /s (if the setup file is setup.iss in the same directory) or

setup.exe /s f1C:\Setup\other.iss if the file have a different name or located somewhere else.


Handheld Software Installation :


For the handheld Software, it's an exe that will extract an MSI on your C:\ so you can use this msi. Modify the msi (or add an mst) and change the AgreeToLicense property to Yes and change the CoutrySelection property to what you want.


If you want users to have a default configuration, you just need to copy three XML files. Here are the steps :


- Run the DesktopMgr.exe with this command line :  DesktopMgr.exe /r

You will now be in record mode until you exit the software. You need to change all the options you want, and exit when everything is done.

- It will creates 3 xml files (DefaultsRedirectorSettings.xml, DefaultsIntellisync.xml and DefaultsDesktop.xml)in the folder where the software was installed. (C:\Program Files\Research In Motion\BlackBerry).You need to copy those 3 files at the same location of the setup.exe so it can find it for the next installation


Note : make sure that isscript.msi is installed on the machine.

According to InstallShield, AdminStudio 6's SmartScan (ability to convert InstallScript/MSI installers to pure MSI installers) does not work properly with the RIM Blackberry InstallScript/MSI installer.


I have seen problems when trying to use a space character " " anywhere within the path or file name of the answer file.  Setup.exe /s /f1C:\Temp Folder\setup.iss would NOT work.  Setup.exe /s /f1C:\TempFolder\setup.iss would work.
