1. Start dt346.exe and press Install. Afterwards open the

      save location and rename it to Daemon-Tools 3.46.0.msi.

      Cancel the Installation of Daemon-Tools.

2. Start Orca and load the file UISample.Msi from the

      Installer SDK.

3. In Orca, go to the menu TablesÂ¦Export Tables... and

      export the table AvtExecuteSequence .

4. Load "Daemon-Tools 3.46.0.msi" to Orca.

5. In Orca, go to the menu TablesÂ¦Import Tables... and

      import the file AdvtExec.idt.

6. Save, That's all! With this, DT will be installed with

      it's default configuration.

      This was tested with GPO-install with "advanced

Note

DAEMON Tools V3.47 using InstallShield DevStudio


1.) Extract the .msi file as described above and open with DevStudio.


2.) On the left side of the screen click on "Custom Actions".  A middle pane will open up.  Click on "DIRCA_TARGETDIR".


3.) You will notice on the right hand most pane a list of properties.  The second one "Property Value" you can edit in case you don't want Daemon Tools to install to it's default directory.


4.) Back to the left hand most pane, click on "Shortcuts".  You can delete the shortcut created on the desktop if you want (I hate messy desktops).


5.) Again, on the left hand most pane, click on "Property Manager".  The very bottom property line says "Click here to add a new item" in light grey type.  Click there and add the name "REBOOT" (all caps and without quotes).  For the value enter in "ReallySuppress" (without quotes and I think may be case sensitive).  This will allow you to install with the /qn switch or via Group Policy with the reboot dialog box appearing.


Tested and verified as working on Windows XP SP2 standalone and via GPO to Windows XP SP2 in a Windows 2000 SP4 domain.
