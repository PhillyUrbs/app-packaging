Note

I imported the following registry lines to prevent the user from having to register and prevent the license.rtf to come up for verification when running the app for the first time.

___________________________________________________________


Windows Registry Editor Version 5.00


[HKEY_CURRENT_USER\Software\Corel\Paint Shop Pro\10\CorelReg\PR10]

"Schedule"=hex:10,00,00,00,01,00,02,00,ff,ff,ff,ff,ff,ff,01,00

"StartDate"="1143141920"


[HKEY_CURRENT_USER\Software\Corel\Paint Shop Pro\10]

"ShowEULA"=dword:00000000

___________________________________________________________

Note

We got some problems after the installation of Photoshop.

When we've installed it, it shows 2 screens by starting the application. First an agreement, and the second one is a registering note.


Because we offer this package by Zenworks it's not comfortable for us that these screens come up after the installation.


To loose the agreement rules change the following register setting:

HKEY_LOCAL_MACHINE - Software - Corel - Paint Shop Pro - 10

Change ShowEula Reg_Dword value from 1 to 0.

In Admin studio delete the Reg_Dword and create a new one. And set the same value.


If you don't delete the origional setting, it won't push the changes.


To get lost of the Register window change the following settings:


HKEY_CURRENT_USER - Software - Corel - Paint Shop Pro - 10 - CorelReg - PR10

create a REG_SZ with the name LastLaunch. The Value Data can be set to 1141028747

Create a REG_SZ with the name StartDate. The Value Data can be set to 1141033136

Create a REG_BINARY with the name Schedule and set the following Value data: 10000000010001000700FFFFFFFF0100


Now the the window wouldn't appear anymore, but if you change the date of the computer it will come up again. So you also need to create the following REG_DWORD "RegistrationSubmitted" and set the value to 1.


Now it wouldn't come up again.


In Admin studio you need to create CorelReg and PR10 in the Register.



Hope it will help you guys!!!

Command Line

Here's the installation I used:


Setup.exe /v"LAUNCH_REGISTRATION=0 UPDATESERVICE=0 APPLAUNCH_REGISTRATION=0 PSP_YAHOO=0 PSP_READMESHORTCUT=0 PSP_RELEASENOTESSHORTCUT=0 PSP_CREATEQUICKLAUNCH=0 PSP_UNINSTALLSHORTCUT=0"


This results in the Registration process not occuring, the Update Service does not install, the user is not given the option to install the Yahoo Toolbar, no QuickLaunch Icon, and no extra shortcuts in Start/Programs, just the application shortcut.  Serial Number is still required.  This is not a silent install.
