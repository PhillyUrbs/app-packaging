Note

The Office 2007 Compatibility Packs allow Office 2003 and XP users to Open, Edit and Save documents, workbooks, projects, presentations, etc generated with Office 2007 products.


To obtain the converters for Word, Excel and PowerPoint 2007 download the FileFormatConverters.exe from Microsoft


The converters can be installed using either the .exe or by extracting the MSI.


Silent Install Using FileFormatConverters.exe:


        FileFormatConverters.exe /quiet


Extracting the MSI:


        FileFormatConverters.exe /extract:c:\temp

Note

There are switches you can use to extract the MSI and source files, then you can create a Transform for the MSI with Wise Package Studio. Within the Transform, set to install per Machine and never reboot and you will not have the issue with the restart window appearing after installation.


Here is the command line option to extract the MSI and source files:

FileFormatConverters.exe /extract:Path where to extract files

Note

In case anyone else is still using this tool, one thing I will mention that affected my deployment of this is that the exe's /extract option is not very helpful in an unattended, scripted deployment.  Adding /passive helps, but it still waits for the user to click OK.  Depending on how you are deploying this, it may be a problem.  My deployment tool waited for the process to close before it continued, so it just hung.


One way I tried to get around it is this:

FileFormatConverters.exe /extract:\file\path\ /passive && ping 127.0.0.1 -n 15 && taskkill /F /IM fileformatconverters.exe

To begin extraction, wait 15 seconds, and kill the process.  But even that didn't work with my tool, even though it worked from the command line.  So instead I just bypassed this and made a 7zip self-extracting exe.


The other entries here were helpful and apart from the above, this was a smooth deployment.
