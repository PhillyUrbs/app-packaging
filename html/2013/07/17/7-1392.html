Note

Didn't have any problems with the 7 SP2 vendor MSI, at least installing. Created a transform using the tuner, then tweaked it a little to turn off the tip of the day, among other things. Best way to handle setting options that the Tuner doesn't include is to install it on a clean workstation, tweak to taste, then go back into HKCU\Software\Novell\GroupWise\Client and copy over whatever registry settings are there that weren't there before. For instance, GWTIP is a new key created when you log in and turn off the tip of the day; the Show DWORD is set to 0. Putting that in your transform will stop that from appearing for new users or the installing user when you install the program. Another nice feature is that the SP2 MSI contains the installer for Windows Messaging, which means you don't need to check for that before you install the actual MSI.

Note

Found the vendor msi to be a complete waste of time.  Too many bugs:


- Installing all available languages support will not install all the english language files (eg gwmxl1us.dll and gwmim1us.dll), so the app defaults to Arabic


- App will not run properly when logging in as a second or third user.  Tries to load the language file for the system default language rather than that specified in the registry (for Australia gwmxl1OZ.dll rather than gwmxl1US.dll).  As far as I'm aware, the OZ version of the file does not even exist anywhere.


- Under certain conditions, an "Internet Directory Accounts" dialog will appear during a silent install.  Not exactly silent.


- Some critical files do not install, eg gwxis10.dll



As a result of all this mucking around, I found the best way to deploy the app is via your own snapshot.  Don't bother trying to deploy the vendor msi in a large organisation - you'll only encounter lots of issues during testing/deployment.
