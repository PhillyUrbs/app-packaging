    Note
    Before installing the client, you need to ensure that the server is first updated to the latest version, because if you package the client, then download, install and activate the VMWare Update Manager Client from Plugin Manager, you will receive an error:

    "Could not load file or assembly 'VimSoapService.25, Version=2.5.0.0, Culture=neutral, PublicKeyToken=null' or one of its dependencies. The system cannot find the file specified."

    This is because the plug-in is being downloaded from the server and is an incompatible version.

    Once the server has been updated, run the vi-client.exe and grab the .msi out of its subfolder in %temp%. Install that, and create a transform. Then, launch the client, go to Plug-in Manager, and download the two plug-ins, grabbing each .msi out of its subfolder in %temp%. You can create transforms for each plug-in, but AdminStudio Tuner would only allow me to do this if the client was installed first.
