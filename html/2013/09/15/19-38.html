Note

You cannot deploy AutoCAD 2010, 2011 or 2012 with SCCM unless DirectX is install ahead of the package. You can wrap both in a .BAT or .CMD or push a separate .MSI (you have to make that yourself) of the DirectX components before or SCCM will report "success" when it fails.  This is documented by Autodesk.

Command Line

Install an Adminimage with SCCM 2007:


Step1:

"AdminImage\Setup.exe" /w /qb /i "AdminImage\Image acad 2011 32bit.ini" /language de-de


Step2:

"install.bat"  (include: Update 1.1 >>  VBA_Enabler >> Raster Design 2011)


msiexec.exe /p "ServicePacks\AutoCAD2011Update1.1.msp" /qb! /norestart /log "C:\logs\AutoCad_2011_Update1.1_install.log"


msiexec.exe /qb! /i "ServicePacks\VBA_Enabler\AcVbaInstaller.msi" /norestart /log "C:\logs\AutoCAD_2011_VBA_Enabler_install.log"


"ServicePacks\Raster_Design_2011_OE_32_deu.exe" /i /q

Command Line

SCCM 2003:

Once you create the deployment using the AutoCAD disc, create a install.cmd and then copy the below into the cmd file:


%~dp0AdminImage\Setup.exe /W /Q /I "%~dp0AdminImage\AutoCAD 2011.ini" /language en-us


regedit.exe /s "%~dp0DisableAutoUpdates.reg"

regedit.exe /s "%~dp0DisableMC3Registration.reg"



*NOTE* Where i have labelled 'AutoCAD 2011.ini' replace this what you titled your ini file (which is found in the AdminImage folder. Also the reg keys are not required but i use this in my deployment to disable auto updates and disable the registration screen
