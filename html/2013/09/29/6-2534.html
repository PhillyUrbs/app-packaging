Note

Should you receive the following error when attempting to install WMDC 6.1 MSI on 64 bit Win7 machines deployed via SCCM, follow the instructions below to fix Microsoft's MSI


DIFXAPP: ENTER: InstallDriverPackages()

DIFXAPP: 'CustomActionData' property 'DIFxApp Version' is 2.1.

DIFXAPP: 'CustomActionData' property 'UI Level' is 3.

DIFXAPP: 'CustomActionData' property 'componentId' is {CADBEB57-D2CF-4CD6-8E85-F688DF3670AC}.

DIFXAPP: 'CustomActionData' property 'componentPath' is C:\Windows\WindowsMobile\Drivers\RNDIS\.

DIFXAPP: 'CustomActionData' property 'flags' is 0x6.

DIFXAPP: 'CustomActionData' property 'installState' is 2.

DIFXAPP: 'CustomActionData' property 'ProductName' is Windows Mobile Device Center Driver Update.

DIFXAPP: 'CustomActionData' property 'ManufacturerName' is Microsoft Corporation.

DIFXAPP: INFO: opening HKEY_USERS 'S-1-5-18\Software\Microsoft\Windows\CurrentVersion\DIFxApp\Components\{CADBEB57-D2CF-4CD6-8E85-F688DF3670AC}' (User's SID: 'S-1-5-18') ...

DIFXAPP: INFO:  ENTER:  DriverPackageInstallW

DIFXAPP: INFO:  RETURN: DriverPackageInstallW  (0xA)

DIFXAPP: ERROR encountered while installing driver package C:\Windows\WindowsMobile\Drivers\RNDIS\wcerndis.inf

DIFXAPP: InstallDriverPackages failed with error 0xA

DIFXAPP: RETURN: InstallDriverPackages() 10 (0xA)


1. open the msi package in Orca

2. goto the Binary table

3. delete the DIFxApp.dll and DIFxAppA.dll row

4. get the latest version of those two files from Win7DDK or WiX or any other source

5. add the two rows again with the new files

6. save the msi and close Orca

7. done, now it should install without any problems


Credit to Manuel Meitinger for the original fix.  We tested his fix and it works.

Note

Extract msi-Files via commandline as mentioned above (xxx.exe /c) or via WinRAR (WinRAR is better, as you can extract x64-files on x86 machines with it).


Use msi-Files directly for setup - it works without problems.


To speed up first connection of a new device, call the .msi-files located in

%Windows%\WindowsMobile\Drivers\RNDIS\wmupdate.msi

%Windows%\WindowsMobile\Drivers\Bluetooth\wmupdate.msi

%Windows%\WindowsMobile\Drivers\Serial\wmupdate.msi

and execute them (unattended by using switch "/qb-!" or similar.


Users have to accept EULA when connecting WMDC and their device for the first time.

Prevent this by creating a key under

"HKCU\Software\Microsoft\Windows CE Services" with name "EulaAccepted" and from type "REG_DWORD" and set its value to 100735797.


WMDC provided by MS is MUI compatible, so you can use the same file(s) for any language.

Note

Once extract these files with the /c command.  You will get two files.

1.drvupdate-x86.msi
2. driver_setup.exe

If you install both of these, the installation will not work because Microsoft is doing sometihng in the wrapper.

You'll have to type in the command:  driver_setup.exe /i drvupdate-x86.msi

This works fine, but I tried to add the slient swtich /qn and /qb-! but I could not get this to run silently.

The drvupdate-x86.exe /Q will work too, but the UI show the cancel button, not ideal if you're tyring to pacakge this.  Just deliver this software with the /Q thorugh SCCM and make sure the program you create in SCCM is set to "hidden" and you'll be fine.  Same thing for the 64 bit version of this.
