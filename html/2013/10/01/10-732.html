Note

MSI Sits in Disc 1 \Premiere Elements

Properties I set are

AgreeToLicense (yes)

COMPANYNAME

SERIALNUMBER

USERNAME


I used a MST to set these for a Group Policy deployment.


I also imported the following reg file into the MST to stop it prompting for registration on first run


[HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall\{6CCDF4E6-D2AE-4DD8-80FD-F9AFF951AEAE}]

"EPIC_REGS_TYPE"=dword:00000004

"EPIC_REGS_STATE"=dword:00000002

"EPIC_REGS_LANG"=dword:00000000

"EPIC_REGS_COUNT"=dword:00000000

"EPIC_REGS_DATE"="2005-03-10 11:02"

Security

The Windows Media Player 9 Runtime player requires more rights than a User Profile provides.  Change the CustomAction Type of each Action listed to Type 3073 from 1025 to move the deferred action to deferred action system context.  As long as the process running has enough rights the user launching the application will be able to install the package.


CustomAction Table:

RunWMAInstaller_DE

RunWMAInstaller_EN

RunWMAInstaller_FR

RunWMAInstaller_JP

CopyReadme

MainConceptSelfReg

PluginFilesSelfReg

PluginFilesSelfUnreg

FixUninstallKeyPermission
