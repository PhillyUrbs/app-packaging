Note

You will get the install source as a bundled Exe like xACF_NW04_SP19.exe.


Run xACF_NW04_SP19.exe /detach=xACF.cab

Unpack the resulting xACF.cab and you have the install source.

Note

Bews is right but you must uninstall and re-install if the version of Adobe Reader changes as well.


If you try to just re-install over top, the SAP installer will just see it's already installed and exit.


You can install this even if Adobe Reader is running. You just have to restart Adobe Reader to use the ACF plugin.


BTW: Shows up as SAP forms in Adobe Reader Help menu->About Adobe Plug-Ins



CAVEAT: If you have Acrobat Pro/Std(any version) after/before Adobe Reader (any version), the SAP plugin-based forms cease to function.

Could not resolve by just unchecking the "Display PDF in Browser" box in Acrobat's preferences

or changing file associations.


The solution was to uninstall and re-install Adobe Reader with an MST created using the "Adobe Customization Wizard"


In the "Adobe Customization Wizard"

-Installation options:

-select "Make Reader the default viewer is both Acrobat and Reader are installed"


<Customize more if you want...>


-Transform Menu ->  Generate Transform

Save at ADOBE_READER_8_1_3.MST




You *might* also be able to forgo the MST route and use the following property in the command line:

IW_DEFAULT_VERB=Read
