For Silent Uninstall (CS5 Applications):

=========================================

1) Extract the "Adobe Captivate CS5" vendor media and look for "deploy" directory.

2) Copy "remove.xml" file from ...\Adobe Captivate CS5\deploy\ to your working share.

3) Run this command for silent uninstall:

"C:\Program Files\Common Files\Adobe\OOBE\PDApp\DWA\Setup.exe" --mode=silent --deploymentFile=<your working share path>\remove.xml

How to disable Adobe Product Improvement Program in Captivate CS5.

Creating a reg file i.e APIP.reg with content as :

****************

Windows Registry Editor Version 5.00


[HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Adobe\APIP]

"enabled"=dword:00000000

**********************

then add it to the registry

How to disable and grey-out updates in Adobe products


Create a file called "AdobeUpdaterAdminPrefs.dat"


****************

<?xml version="1.0" encoding="UTF-8" ?>

<Preferences>

<Suppressed>1</Suppressed>

</Preferences>

*********************


then copy AdobeUpdaterAdminPrefs.dat to \%Program Files Common%\Adobe\AAMUpdaterInventory\1.0

Note

After installing on Windows 7, there is a box that comes up on first use about Administrator rights.  This is created by launching the advertised shortcut.  To suppress this warning box, you'll need the have an existing objects.cpo file, stored in AppData\Local\Adobe\Captivate 5.5\objects\objects.cpo. 

If I just create that objects folder before running the advertised shortcut, it gets overwritten with the settings.  However, if the captivate 5.5 folder is copied in before running Captivate, that suppresses that warning box and the advertised shortcut doesn’t need to create the folders.
