Command Line
to silent install Windows Live essential use switchs like this :

wlsetup-all.exe/AppSelect:<ApplicationID>,!<ApplicationID> /q

/AppSelect:<ApplicationID> select features you want to install

!<ApplicationID> are features that should not be installed

/q mean silent installation

<ApplicationID> can be :


ALL (Full live essential)
Mail
PhotoGallery
OLC (Outlook Connector)
Messenger
Writer
MovieMaker
Companion (Messenger Companion)
FamilySafety
wlsync (Live Mesh)
Bingbar
idcrl (Live Sign in Assistant)


exemples :

Installing live mesh and live writer only :

wlsetup-all.exe/AppSelect:wlsync,Writer /q

Installing full package except bingbar and messenger :

wlsetup-all.exe/AppSelect:ALL,!messenger,!Bingbar /q


some useful additional options :

/log:<filename> logging your installation

/noMU disable Windows Update

/noHomepage prevent homepage to become Msn.com

/noLaunch prevent Messenger to launch just after installation as finished.

/noSearch prevent your default search engin to be replaced by Bing.com

Note: The above installation sequence should be followed for successful installation of Windows Live Messenger. If this sequence is not followed, the installation fails.
It is noteworthy that, 16 MSIs and MSP are very much needed for the installation of only one feature Windows Live Messenger out of the suite.

Installing Microsoft Silverlight - Disable Update :-
After Installing the patch (MSP) for Microsoft Silverlight, we have a shortcut in start menu which opens the Silverlight control panel.
Following per user registry are added to disable auto update in the above mentioned Silverlight control panel.
[HKEY_CURRENT_USER\Software\Microsoft\Silverlight]
"UpdateMode"=dword:00000002
"DRMEnabled"=dword:00000000

Installing Microsoft Application Error Reporting:-
The sequence number for this MSI is 7.
Microsoft Error Reporting is installed with following command line parameters:-
Msiexec /i Error_Reorting.msi PPGUID={00000000-0000-0000-0000-000000000000} REBOOT=ReallySuppress MSIRESTARTMANAGERCONTROL=Disable MSIFASTINSTALL=7 ROOTDRIVE=C:\ CLIENTUILEVEL=3 MSICLIENTUSESEXTERNALUI=1

Installing Windows Live Messenger:-
The sequence number for this MSI is 13. This is the main MSI that installs Windows Live Messenger executables.


To disable thesr windows, add following per user registry keys.
[HKEY_CURRENT_USER\Software\IM Providers\MSN Messenger]
"UpAndRunning"=dword:00000000
[HKEY_CURRENT_USER\Software\Microsoft\Windows Live\Common]
"FREDate"=hex:bc,83,40,ed,ff,6a,cb,01
"SettingsVersion"=dword:00000004
"SQM"=dword:00000001
"TOUVersion"="15.0.0.0"
"UserLanguage"="en"

Un-installation steps:-
For un-installation of MSIs, the sequence is reversed as that for installation.
Also note that, MSP is not removable during un-installation.To do so, complete un-installation of Microsoft Silverlight that is already present on the build is needed.
Microsoft Error reporting will also not un-install if Microsoft products such as MS Office is already present over the build. After installation of MS Error Reporting, it gets associated with Office implicitly.

Transform is applied only on Windows Live Messenger (No. 13th MSI in the sequence given above), rest all other customization such as :-
Disable Autoupdate for Silverlight are carriedo out in install script and with active setup(Per User settings)

Modifications
Messenger.msi is missing its installation tables so adding via GPO doesn't work without some modifications.
Open up Messenger.msi in Orca
Create a New transform

Open up another instance of Orca
