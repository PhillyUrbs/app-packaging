Deployment Tip Summary

With version 8.5 of the DYMO software, most of the prior version tips did not work. I contacted DYMO and they sent me a file that changed it so registration did not pop up and the updates are disabled. Open Notepad Copy the following into the document false True True True False False True True True True True flase False Save the file as DLS.exe.config In Windows 7 Save it in C:\Program Files (x86)\DYMO\DYMO Label Software In Win XP save it in c:\program files\Dymo\Dymo label software Run the normal Dymo executable and it will come up with no prompts for registration or updates. http://download.dymo.com/Software/Win/DLS8Setup.8.5.0.exe This may or may not work on earlier or later versions of the software, but it does work for sure on 8.5.
or cancel

You can use following registry key to avoid it.

[HKEY_CURRENT_USER\Software\DYMO\DLS8]"Culture"="en" "PreviousVersion"="8.3.0.1242" "Location"="US" "Initialized"=dword:00000001

I tried the method mentioned above but didn't work for me so i found out that when we click on "Register" it takes you to the registration website and even if you dont register and close the website, it generates a folder "Sanford,_L.P" in %USERPROFILE%\AppData\Local. This folder includes different files for 64-bit machine and 32-bit machine. This folder can later be used for all of your machines in the organization to get rid of registration popup. Enjoy....

One of them is named "DLS.exe_Url_q0dsxht4oe0yovrhtt0kppdqugmsjweh" that suppresses the registration process for 32-bit machine and the other is "DLS.exe_Url_oytsydlkqagtfrul0qdbmkyn2y0toxat" that suppresses registration process for 64-bit machine.

Note
Prevent registration wizard Dymo Label v8.3:

1. Open the file C:"|Users|username|AppDataLocalSanford,_L.PDLS.exe_Url_q0dsxht4oe0yovrhtt0kppdqugmsjweh|buildnumber|user.config
Be aware that username, the string q0dsxht... and buildnumber might be different on your computer.
2. Change the value on the node "RegistrationCheckCount" from 0 to 1 (the value between the "value" tags)
3. Save and close the file.

The registration wizard will no longer pop up.

Note
Got this info from DYMO Helpdesk:

Solution for Windows (Dymo Label version 8 or later) 1. Open the file C:"|Users|username|AppDataLocalSanford,_L.PDLS.exe_Url_q0dsxht4oe0yovrhtt0kppdqugmsjweh|buildnumber|user.config
Be aware that username, the string q0dsxht... and buildnumber might be different on your computer.
2. Change the value on the node "RegistrationCheckCount" from 0 to 1 (the value between the "value" tags) 3. Save and close the file.
The registration wizard will no longer pop up.

For Windows XP you should replace "Users" with "Documents and Settings",

"AppDataLocal" should be replaced with "Local SettingsApplication Data".

The path should then look similar to this, C:Documents And Settings|username|Local SettingsApplication DataSanford,_L.PDLS.exe_Url_q0dsxht4oe0yovrhtt0kppdqugmsjweh|buildnumber|user.config

Solution for Mac
1. Go to the folder /Users//Library/Preferences 2. Open the file com.dymo.dymoreg.plist (e.g. with TextEdit) 3. Look for the key called UseCount and change its value to 4 4. Save the file.
The registration wizard will no longer pop up.

Solution for Windows 2003 Terminal Server

Please note: The DLS software is not developed for and has not been tested on Terminal server systems. In case below suggestion does not solve (file is in a folder on a location that is no part of the roaming profile) the issue we unfortunately have no other solution available.

Modify user.conf.xml (in C:"|Users|username|AppDataLocalSanford,_L.PDLS.exe_Url_q0dsxht4oe0yovrhtt0kppdqugmsjweh|buildnumber|user.config
by adding at the bottom of the text (below /DYMO.DLS.Properties.Settings):

{setting name="IsUserRegistered" serializeAs="String"}
{value}True{/value}

(change the {} brackets into valid brackets)

and they told me to block Dymo with the firewall to block the Auto-Update function.
