Note

problem with the command lines are additional users on the same computer, won't see the shortcuts nor able to run the program. so i took the files and build an msi, without any registry keys. three registry lines are needed per user in order to fix the "A required resource was unavailable" error.

REGEDIT4

[HKEY_CURRENT_USER\Software\j2 Global\eFax Messenger 4.4\J2GSDK]
"InstallAppID"="{DF6DA606-904D-4C18-823F-A4CFC3035E53}"

[HKEY_CURRENT_USER\Software\j2 Global\eFax Messenger 4.4\J2GSDK]
"InstallVersionNew"="4.4"

[HKEY_CURRENT_USER\Software\j2 Global\eFax Messenger 4.4\J2GSDK]
"UserInstallTime"=hex:b7,f5,db,4f,

Command Line
Download the latest version (tested on version: 4.4.0.514) and run: msgrplus.exe /extract

This will create "eFaxMessenger4.4.ini" and a directory called "C3035E5" in the same folder msgrplus.exe was executed from.

Then run: msiexec /i eFaxMessenger4.4.msi /quiet INSTALLLEVEL=1

INSTALLLEVEL=1 should only install the base application if there are any other added apps bundled with the install. If only there were a way to keep from installing the startup option...

Command Line
Not sure if the INSTALLLEVEL option works as referenced... There are not that very many deltas. But to answer the question on the Startup ICON... This is the commandline I use.

msiexec "eFaxMessenger4.4.msi" /qb REBOOT=ReallySuppress INSTALLLEVEL=1 STARTTRAY=0

I always do the REBOOT options just as a precaution. Anyway, that prevents the startup icon, STARTTRAY=0
