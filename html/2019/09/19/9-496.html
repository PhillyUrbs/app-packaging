Note
There are a couple options with this one.
Option 1- Edit ABCPY.INI in the root installation directory. The following options are available.
-----------------
[OEM Install]
SERIALNUMBER=XXXXXXXXXXXX
SERIALNUMBERTRYOUT=
USERNAME=Your Name
COMPANYNAME=Your Co.
INSTALLDIR=
------------------
Run a MSIEXEC /I "Path to Adobe PhotoShop CS2.MSI" /qb

Option 2 - My Recommendation
Run the "Adobe PhotoShop CS2.MSI" through installshield tuner. I do not suggest changing any of the "Features" options, as Photoshop should just install everything it needs.

Go down to Properties and edit:

AgreeToLicense: Change to Yes
ISCHECKFORPRODUCTUPDATES: I change to "0" because if there are updates, we do not want those automated on a locked down PC.
REBOOT: Yes (Forces users to reboot every once and a while)
SERIALNUMBERTEMP: Your Serial #.

Hope this helps.
-Luke

Note
Standard MSI, except that it does not call the other MSI's it also needs for full install. I installed by using this sequence:

1. Adobe Photoshop CS2\Adobe Photoshop CS2.msi TRANSFORMS="Adobe Photoshop CS2.mst"
2. Adobe Photoshop CS2\commonfilesinstaller\Adobe Common File Installer.msi
3. Adobe Photoshop CS2\bridge\Adobe Bridge 1.0.msi
4. Adobe Photoshop CS2\help center\Adobe Help Center 1.0.msi
5. Adobe Photoshop CS2\stock photography\Adobe Stock Photos 1.0.msi

In the transform, the following actions are done:
1. Disable Adobe updater by adding the following key:
HKEY_LOCAL_MACHINE\SOFTWARE\Adobe\Updater

Then add a Dvalue called: Enterprise and enter in a value of 00000001

2. Suppress registration dialog box:
Add following 4 keys under (Adobe site is off a little bit)
HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall\[ProductCode]
a. "EPIC_REGS_TYPE"=DWORDValue00000004
b. "EPIC_REGS_STATE"=dword00000002
c. "EPIC_REGS_LANG"=dword00000000
d. "EPIC_REGS_COUNT"=dword00000000

Note
If you are needing to extract the .msi for a network install using shortfilenames. You will need to open the .msi with an editer such as Orca from M$ and edit the following lines in the "Directory" Table.

Search for this: "Photoshop 9.0:Photoshop 9.0" change to: "Photos~1:Photos~1"

Search for this: "9.0.0" and change the two (2) entries to this: "9~2"

It will then extract fine in shortfilenames for the network install. The tips from the guys above are also very handy, especially to disable the auto updater.

Note
MAIN INSTALLATION:

My silent installation was working with /qr, but would always fail with /qn. I was hoping that I could get away with just using the .msi, .mst (with serial number etc) and .cab files, and hadn't copied my customized abcpy.ini file. Turns out that you need this abcpy.ini file in the same directory of the msi regardless for a totally silent installation to work.

PATCH:

The 9.0.1 patch is now available from:

http://www.adobe.com/support/downloads/detail.jsp?ftpID=3335

(this is the multilingual patch which appears to be the only one for english installations).

To install silently, extract the contents to a folder, then run "setup.exe -R". This will create a file called setup.iss in %windir%, which you can then copy to the same directory as the setup.exe. You can then run setup.exe -s to install silently and remove the "Setup finished" prompt.

NOTE: The -s isn't case sensitive but the -R is! (ie. if you use -r instead of -R, it will not create the setup.iss file).

Note
A Custom Action in "User Interface" and "Execute Immediate" are overwriting the SERIALNUMBER property with data from the Abcpy.ini file.

To avoid the SERIALNUMBER property to be overwritten, outcomment or remove the following Costum Action from both "User Interface" and "Execute Immediate":
"CA_GetUserInfo"

Note
In order to make the autorepair functionality from windows installer to work with Photoshop CS2, remove the custom action CA_GetUserInfo (as mentioned by MMA on this same page) and make sure to provide the property SERIALNUMBER with the right serial. Remove or edit the existing shortcuts for Photoshop.exe and ImageReady.exe so they are advertised.

Now the auto-repair functionality works.
Apparently the CA_GetUserInfo Custom action which takes registration info from Abcpy.ini doesn't work or isn't used when repairing...

To make the repair work for normal (non admin) users don't forget to remove the Launch Condition that checks for admin rights.

Note
Adobe Photoshop application, upon launching, shows a welcome screen. This can be suppressed by following the steps below.

1. Install the product and launch the application.

2. De select the option 'Show this dialog at startup' on the welcome screen and close the window.

3. Copy the file 'Adobe Photoshop CS2 Prefs.psp' file from %Appdata%\Adobe\Photoshop\9.0\Adobe Photoshop CS2 Settings folder.

4. Include this file in your MST under the same location. Please note that this file should be installed to the current user's profile.
