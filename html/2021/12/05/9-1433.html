    Note
    To get this to install in our environment (Fully locked down XP SP2) we did the following:

    Create .mst with orca or something similar.

    Remove EregInitialize.7F8AD70B_6EFA_4403_9206_16D3E1284FEC from the CustomAction and InstallExecuteSequence tables.

    Change the following in the Property table

    AgreeToLicense=Yes
    PRODUCTUPDATEFLAG=0
    SERIALNUMBER=?????-???-????-????-??

    This will install using elevated privileges and prompt the user to activate and enter S/N on first run.

    If you are getting the error DgnUseDll failed with error code -1 when attempting to load c:\users\Christ~\AppData\Local\Temp\{DDDD90B2-80F2-413A-8A8E-38C5076A7DBA}\mobile.dll into memory or something similar when doing a silent install you will need to install the Microsoft Visual C++ 2005 Redistributable Package (x86) that can be found here first.

    Note
    I tested the installation using both sets of notes listed above. It is definitely necessary to remove EregInitialize.7F8AD70B_6EFA_4403_9206_16D3E1284FEC from both locations to avoid the prompt for Web registration.

    WEBREGISTRATION=0 did not remove the web registration prompt, so I'm not sure what it does, even though I set it to 0 in my production deployment.

    Also, I found that the ISScript1050.msi for version 10.5 is a prerequisite in my environment. I noticed that it was provided in the installation directory of the install CD, which says alot.

    If your installation fails due to the ISScript error, follow these directions or pre-execute the isscript1050.msi:

    http://consumerdocs.installshield.com/selfservice/viewContent.do?externalId=Q108158&sliceId=1

    Note
    Guys totally found a significantly easier way to the install. Simply copy all the files out of the two cds to which ever directory you would like. After completing this go in and edit the setup.ini. All you have to do in there is add your parameters you were passing to the msi.
    Ex:/Liwmo!e+ %TEMP%\dgnsetup.log SERIALNUMBER=XXXX-XXXX-XXXX-XXXX-XX PRODUCTUPDATEFLAG=0 /qb

    Resave over the original setup.ini! Once you do this just call the setup and you are money. Simple is that. Hopefully this helps someone I always feel I am two steps behind you guys in scripted apps. Thanks guys for everything!

    Note

    Yep Warbie's suggestion works for 12.0 too. Changed the setup.ini CmdLine:

    CmdLine=/Liwmoe+ %TEMP%\dgnsetup.log SERIALNUMBER=XXXXX-XXX-XXXX-XXXX-XX PROD_ACT=0 PROD_REG=0 EREG_RUN=0 PRODUCTUPDATEFLAG=0 WEBREGISTRATION=0 DWUSMANAGER=0 /qn

    ...although I don't think it gets you over having to register. I just used the XXX's for serial number - the user then gets prompted to put the serial number in and activate and gets 5 additional uses.
     
    And I think %temp... fails if you run as a user account, you need to put quotes round it as the %temp% variable might have spaces in it; "%TEMP%\dgnsetup.log"

    Note
    Warbie's suggestion to edit setup.ini works great! FYI, The full CmdLine line we're using is:

    CmdLine=/Liwmo!e+ c:\tools\dgnsetup.log SERIALNUMBER=XXXXX-XXX-XXXX-XXXX-XX PROD_ACT=0 PROD_REG=0 EREG_RUN=0 PRODUCTUPDATEFLAG=0 WEBREGISTRATION=0 DWUSMANAGER=0 /qb

    Obviously, replace the SERIALNUMBER X's with your serial number, and if necessary, change the log output destination (%TEMP%\dgnsetup.log didn't work for us, so I've pointed ours to a toolkit folder...)
