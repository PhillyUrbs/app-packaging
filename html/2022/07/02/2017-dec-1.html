    Note
    The command line for the package created on 19 Dec 2017 is

    setup.exe --silent

    Adobe was not not able to provide a working silent install. The above command was guessed by reading pdapp.log

    Installation progress can be seen in pdapp.log

    Note

    I'm using <Setup.exe /sAll /sl "1033"> and have rolled in the latest updates (having already downloaded and saved to the root.    Edit setup.ini in the root folder and under the product section add the name of the file.  e.g.


    [Product]
    PATCH=Acrobat2017Upd1701130059.msp


    If you're looking to do update surpression do the following;

    sc stop adobearmservice
    sc config adobearmservice start=disabled
    reg add "HKLM\SOFTWARE\Policies\Adobe\Adobe Acrobat\2017\FeatureLockDown" /f /v "bUpdater" /t REG_DWORD /d 0
