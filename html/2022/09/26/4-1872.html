    Note
    If you want to use the MSI for installation and customization, do the following:

    - Start LyncSetup.exe
    - When the Microsoft Lync 2010 setup screan is displayed, browse to C:\Program Files (x86)\OCSetup (or C:\Program Files\OCSetup for 32-bit) and copy the Lync.MSI
    - Create a transform for Lync.msi
    - Add the property USEMSIFORLYNCINSTALLATION with value #1

    Now you can install the msi without using the setup program.

    Note
    Show Lync in foreground when it starts:

    You'll need to set the DWORD registry key "AutoOpenMainWindowWhenStartup" HKCU\Software\Microsoft\Communicator to the value 0.

    Minimize to notification Area:

    You'll need to set the DWORD registry key "MinimizeWindowToNotificationArea" HKCU\Software\Microsoft\Communicator to the value 1.

    Command Line
    For installing Lync 2010, using the LyncSetup.exe, the following switches can be used:

    Install options:
    /Install - Installs all files
    /Uninstall - Removes all files

    Display options:
    /Silent - No UI displayed

    For example:
    LyncSetup.exe /Install /Silent will install Microsoft Lync 2010 silent with default settings.

    Note

    If you do an install with the MSI, follow the above instructions but also note that when doing an installation with the MSI, the folder OCSSETUP isn't created in ProgramFilesFolder. Because of this, every time you start Lync 2010 it will cause a repair because Windows Installer is looking for that folder. To solve this issue, edit the OCSSETUP entry in the Directory-table and set the parent to ProgramFilesFolder and call the folder "Microsoft Lync".

    4
    Note
    Please note that the Property has to be set as follows:

    USEMSIFORLYNCINSTALLATION = "#1"

    The # sign is needed.

    Alternatively you can delete the following row from the LaunchCondition table:
    OCSETUP OR Installed OR USEMSIFORLYNCINSTALLATION = "#1"

    Note
    For the existing note that instructs users to create a transform and add the property USEMSIFORLYNCINSTALLATION (setting the value to 1), this worked for me only after I first made the following registry change:
    HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Communicator
    Add a DWORD value named UseMSIForLyncInstallation and set it to 1.
    After that the MSI installed successfully, using the transform as outlined in meino's existing note.

    Command Line
    Adding the Microsoft MSP silently

    x64
    msiexec /p Lync_x64.msp REINSTALL=ALL REINSTALLMODE=omus /qn

    x86
    msiexec /p Lync_x86.msp REINSTALL=ALL REINSTALLMODE=omus /qn

    If Lync is running ti will shut it down and apply the patch. Then you simply need to launch Lync again.
