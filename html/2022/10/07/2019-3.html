    KACE Managed Installation

    Microsoft Office Professional Plus 2019 / 365

    NOTE 1:  I don't believe Office 2019 is within the Software Catalog yet (Our KACE rep could not find it either) so we installed Office 2019 on 1 PC and deployed it via Software instead of Catalog.

    NOTE 2:  I highly recommend selecting the Local Source option in your Office Customization Tool in Pre-Deployment Step 2 and running the following command to download the install files to a shared software folder: "Setup.exe /download <name of .xml file created in Pre-Deployment step 2>"

     

    Office 2019 Deployment Guide can be found here

     

    Pre-Deployment (NON-KACE):

    1. Download and Extract the Microsoft ODT from HERE to a newly created folder for ease of use later.
    2. Visit HERE to use the Office Customization Tool and download the created .xml configuration file to the same folder as you extracted the ODT to.
    3. Zip the contents (not the actual folder) of the extracted ODT folder including the .xml created using the Office Customization Tool to a compressed file (.zip)

     

    KACE Deployment

    1. On the KACE 1000, navigate to Inventory > Software and find "Microsoft Office Professional Plus 2019 - en-us" in the software list
    2. Upload the .zip file you created in Pre-Deployment step 3 under the :Upload and Associate File" section and SAVE
    3. Navigate to Distribution > Managed Installers and create a NEW Managed Installation
    4. Name your Installation
    5. Choose an Execution Type
    6. Choose Software under "Inventory"
    7. Select "Microsoft Office Professional Plus 2019 - en-us" under "Software"
    8. Leave Associated File as default "Use associated file"
    9. Choose "Override default installation" under Installation Options and enter the following in the "Full Command Line" area:  "Setup.exe /configure <name of .xml file created in Pre-Deployment step 2>"

    10. Choose Systems or Labels to deploy to
    11. Choose notification options
    12. Choose a deployment window
    13. SAVE



    Office 2019 / 365 should now deploy through your KACE 1000!

    Command Line
    1. Download ODT (Office Deployment Tool)
      https://www.microsoft.com/en-us/download/details.aspx?id=49117

    2. Run the ODTool executable and extract file to a folder
      e.g. \\bla\blabla\Office\Office2019

    3. Open extracted location and create a copy of the XML file e.g. config.xml
      Example of config.xml:

    4. Start cmd as admin and navigate to the setup.exe location and run this cmd to download office 2019
      setup.exe /download config.xml

    5. You will see Office folder (More than 2Gb) in the same location that you have chosen in the config file 
      e.g. \\bla\blabla\Office\Office2019

    6. To install office use this cmd:
      \\bla\blabla\Office\Office2019\setup.exe /configure config.xml

    Note

    When creating your custom .xml file using the Office Customization Tool from here https://config.office.com/ be sure to specify the correct path.  I found it easiest to specify an existing network fileshare path that most authenticated users have permissions to.  Paste your files downloaded from the Office Deployment Tool there and set appropriate permissions so Kace (or the user/account it is running as) can access these files.  ODT is available here: https://www.microsoft.com/en-us/download/details.aspx?id=49117 

    Once you have your completed XML, zip that along with setup.exe and create a scripted install.  Be sure to make note of the name you use for your config file like "configurationx86.xml"

    Down under dependencies, click the plus for new dependency and upload your previously created zip file. 

    Under Task 1 click add under the on success section, and click launch a program and enter the following: 

        Directory: $(KACE_DEPENDENCY_DIR) 

        File: setup.exe

        [v] Wait for completion

        [  ] Visible - leave un-selected for silent install as to interrupt users, or check for testing purposes. 

        Parameters: /Configure .\configurationx86.xml    where <configurationx86.xml> is the name of your previously created xml file. 


    This process has Kace handling the installer and XML, but the fileshare serves the files.  I attempted a few tries to do a managed install through Kace where SMA handles everything (easier on permissions) but have not been successful yet. 


    Anyone else have success deploying Office 2019 or click-to-run?

    Command Line


    1.  I downloaded the ODT (Office Deployment Tool)  https://www.microsoft.com/en-us/download/details.aspx?id=49117

    2.  Run the ODTool executable and extract file to a folder  e.g. \\share\Office\Office2019

    3.  If not using Office 365, delete those (not needed)

    4.  Changed the configuration .xml file to this:  (We use KMS so PID is not needed)

    -         <Configuration>
    -         <Add OfficeClientEdition="64" Channel="Perpetual2019">
    -         <Product ID="ProPlus2019Volume">
              <Language ID="en-us" />
              <ExcludeApp ID="Groove" />
              <ExcludeApp ID="Lync" />
              <ExcludeApp ID="OneDrive" />
              <ExcludeApp ID="Outlook" />
              </Product>
              </Add>
              <Updates Enabled="TRUE" Channel="PerpetualVL2019" />
              <Display Level="Full" AcceptEULA="TRUE" />
              <RemoveMSI All="True" />
              </Configuration>

    5. Created a scripted named "Install.cmd" with this:  setup.exe /configure .\configuration.xml

    6.  Zipped up only these 3 things:  setup.exe, configuration.xlm and Install.cmd.

    7.  In Managed Installation I change to Override default installation calling "Install.cmd"
