    Note
    The overall method that has worked for me is the one described by Adobe here; http://kb2.adobe.com/cps/861/cpsid_86151.html - however there are some changes that you need to make for a truly silent installation and to avoid certain annoyances for the end-users when they first use the program.

    \\your.server.name\yoursharename\oem.exe /UL1033 /V"SERIALNUMBER=1234-5678-9012-3456-7890-1234 COUNTRY=242"

    Note the addition of the Country setting. 242 is the code for UK, check the 'Notes' section below for a list of the country codes (these can be found inside the MSI package). The Country setting is different to the install package language (1033 for English, others listed on the Adobe page).

    Other changes from the official method:
    Change your application.xml.override file in the root folder to match the following to fix EULA popups...

    <?xml version="1.0" encoding="utf-8"?>
    <Configuration>
    <Payload>
    <Data key="EULA">Suppress</Data>
    <Data key="Registration">Suppress</Data>
    <Data key="Updates">Suppress</Data>
    </Payload>
    </Configuration>

    Note
    Here are the contents of the Abcpy.ini file that I'm using. It's as recommended by Adobe in their silent deploy article...

    [OEM Install]
    SERIALNUMBER=
    INSTALLDIR=
    INSTALLTEMPLATES=1
    DISABLEEMSFEATURES=1
    DISABLEOLSFEATURES=1
    WATCHSERVICE=0

    The 'Templates' folder in the root of my deployment share contains the full contents of the extra 'Contents' disk for the additional themes. Leave INSTALLTEMPLATES=0 and forget about creating a templates folder if you aren't installing the additional extras.
