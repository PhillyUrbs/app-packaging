    Command Line

    Media provided appears as so:


    Ideally, the following should be performed on a snapshotted cleans VM.

    Clean means clean- no other software installed! This package has pre-requisite software, so for recording the answer file, you want it to detect that it's needed. 

    Steps to build:

    1. Copy contents of the /SAS Software Depot/JMP/JMP/15_2 folders (Containing the setup files) to a local folder such as C:\TEMP
    2. From a command prompt, execute the installer in recording mode 
      c:\temp\setup.exe -r -f1"c:\TEMP\silent.iss"
    3. Click through the installer, making sure to check the correct settings, such as installing pre-requisites if needed. 
    4. Once the installer is finished, and JMP is installed, launch the software.
    5. Enter your license information.  
      If the license file was applied during the installation, no further action is required. If the license file was not applied during the installation, the user is prompted to select a license file. If the user points to a JMP.PER file, the location of the JMP.PER file is stored. This means if the JMP.PER file is later moved or deleted, the user is prompted to select a license file. This also means that if the JMP.PER file is stored on a network drive, that drive must be available to run JMP. If the user points to a SID file, JMP uses the file to create a JMP.PER file and then prompts the user for a location to save the file.
    6. Ensure that silent.iss was created at the location specified. 
    7. If you have a jmp.per file to authorize the application, include it in the same folder location. 
    8. Zip the contents of the folder that contains the setup files, being sure to include the silent.iss file, and the jmp.per file (if applicable).
    9. In the KACE SMA, Create the deployment [Managed Install] and use the following commands to execute the contents of the ZIP 

    setup.exe -s -f1".\silent.iss"
    copy /y jmp.per "C:\Program Files\SAS\JMP\15"
