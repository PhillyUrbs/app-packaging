    Command Line
    to silent install Windows Live essential use switchs like this :

    wlsetup-all.exe/AppSelect:<ApplicationID>,!<ApplicationID> /q

    /AppSelect:<ApplicationID> select features you want to install

    !<ApplicationID> are features that should not be installed

    /q mean silent installation

    <ApplicationID> can be :


    ALL (Full live essential)
    Mail
    PhotoGallery
    OLC (Outlook Connector)
    Messenger
    Writer
    MovieMaker
    Companion (Messenger Companion)
    FamilySafety
    wlsync (Live Mesh)
    Bingbar
    idcrl (Live Sign in Assistant)


    exemples :

    Installing live mesh and live writer only :

    wlsetup-all.exe/AppSelect:wlsync,Writer /q

    Installing full package except bingbar and messenger :

    wlsetup-all.exe/AppSelect:ALL,!messenger,!Bingbar /q


    some useful additional options :

    /log:<filename> logging your installation

    /noMU disable Windows Update

    /noHomepage prevent homepage to become Msn.com

    /noLaunch prevent Messenger to launch just after installation as finished.

    /noSearch prevent your default search engin to be replaced by Bing.com

    Note
    Windows Live messenger is one of the feature available with Windows Live Essentials 2011 suite.
    Target Operating system : Windows VISTA / Windows 7:- Windows live messenger (Windows Live essentials 2011 suite) is not compatible with Windows XP)

    Source:
    Online installer : wlsetup-web.exe (Approx 1.2 MB). As the name suggest, the installer downloads files from internet during installation.
    This setup files is available at Download wlsetup-web.exe

    Offline installer (Stand alone) : wlsetup-all.exe (Approx 155 MB).This source contains all the files needed during installation.
    Installation steps :As the scope of the document is only limited to Windows Live messenger, we select Messenger feature.
    Proceeding with installation, setup extracts following MSIs and MSP at C:\Windows\Installer\ .These MSIs have random GUID names.

    Below is the list of 16 MSIs and 1 MSP extracted,

    1. Microsoft Silverlight - Update (msp) :- Windows Live messenger requires latest Silverlight update for functioning (Since Silverlight is already present on the build, only the patch gets installed).
    2. Windows Live ID Sign-in Assistant (msi):- The Windows LiveÃ¢Â„Â¢ ID Sign-in Assistant installs the Windows Live ID online provider for WindowsÃ‚Â® 7. Links a Windows Live ID to a Windows 7 user account.
    3. Windows Live UX Platform (msi) :- The underlying frame work required for Windows Live messenger and other Live features.
    4. Windows Live Installer (msi) :- Installs the base files related to Windows Live messenger and other Live features.
    5. MSVCRT (msi) :- msvcrt.dll is a module containing standard C library functions such as printf, memcpy, and cos. It is a part of the Microsoft C Runtime Library. 6. D3DX10 (msi) :- The d3dx10.dll file is one of many files contained in the DirectX software collection. Since DirectX is utilized by most Windows based games and advanced graphics program. Required for Windows Live messenger and other Live features.
    7. Microsoft Application Error Reporting (msi) :- Error reporting functionality provided by Microsoft.
    8. Windows Live SOXE Definitions (msi) :- Definition files for underlying frame work required for Windows Live messenger and other Live features.
    9. Windows Live SOXE (msi) :- Underlying frame work required for Windows Live messenger and other Live features.
    10. Windows Live Communications Platform (msi) :- Underlying frame work required for Windows Live messenger and other Live features.
    11. Windows Live PIMT Platform (msi) :- Underlying frame work required for Windows Live messenger and other Live features.
    12. Windows Live Photo Common (msi) :- Framework for Image related functionality for Windows Live messenger and other Live features.
    13. Windows Live Messenger (msi):- Live messenger executable files.
    14. Windows Live UX Platform Language Pack (msi):- Language pack for Live UX Platform.
    15. Windows Live Essentials (msi) :- Supporting files for Windows Live messenger and other Live features.This MSI is responsible for displaying Ã¢Â€ÂœWindows Live EssentialsÃ¢Â€Â in ARP.
    16. Windows Live Photo Common (msi):- Supporting files for Image related functionality for Windows Live messenger and other Live features.
    17. Windows Live Messenger2 (msi) :-Supporting files for Live messenger .

    Note: The above installation sequence should be followed for successful installation of Windows Live Messenger. If this sequence is not followed, the installation fails.
    It is noteworthy that, 16 MSIs and MSP are very much needed for the installation of only one feature Windows Live Messenger out of the suite.

    Ã¢Â€Â¢ The source also installs Ã¢Â€ÂœMicrosoft Silverlight 4.0.50401.0Ã¢Â€Â as part of installation.
    Ã¢Â€Â¢ Ã¢Â€ÂœMicrosoft SilverlightÃ¢Â€Â and Ã¢Â€ÂœWindows Live ID Sign-in HelperÃ¢Â€Â plugins for Internet explorer are installed.

    Installing Microsoft Silverlight - Disable Update :-
    After Installing the patch (MSP) for Microsoft Silverlight, we have a shortcut in start menu which opens the Silverlight control panel.
    Following per user registry are added to disable auto update in the above mentioned Silverlight control panel.
    [HKEY_CURRENT_USER\Software\Microsoft\Silverlight]
    "UpdateMode"=dword:00000002
    "DRMEnabled"=dword:00000000

    Installing Microsoft Application Error Reporting:-
    The sequence number for this MSI is 7.
    Microsoft Error Reporting is installed with following command line parameters:-
    Msiexec /i Error_Reorting.msi PPGUID={00000000-0000-0000-0000-000000000000} REBOOT=ReallySuppress MSIRESTARTMANAGERCONTROL=Disable MSIFASTINSTALL=7 ROOTDRIVE=C:\ CLIENTUILEVEL=3 MSICLIENTUSESEXTERNALUI=1

    Installing Windows Live Messenger:-
    The sequence number for this MSI is 13. This is the main MSI that installs Windows Live Messenger executables.


    To disable thesr windows, add following per user registry keys.
    [HKEY_CURRENT_USER\Software\IM Providers\MSN Messenger]
    "UpAndRunning"=dword:00000000
    [HKEY_CURRENT_USER\Software\Microsoft\Windows Live\Common]
    "FREDate"=hex:bc,83,40,ed,ff,6a,cb,01
    "SettingsVersion"=dword:00000004
    "SQM"=dword:00000001
    "TOUVersion"="15.0.0.0"
    "UserLanguage"="en"

    Un-installation steps:-
    For un-installation of MSIs, the sequence is reversed as that for installation.
    Also note that, MSP is not removable during un-installation.To do so, complete un-installation of Microsoft Silverlight that is already present on the build is needed.
    Microsoft Error reporting will also not un-install if Microsoft products such as MS Office is already present over the build. After installation of MS Error Reporting, it gets associated with Office implicitly.

    Transform is applied only on Windows Live Messenger (No. 13th MSI in the sequence given above), rest all other customization such as :-
    Disable Autoupdate for Silverlight are carriedo out in install script and with active setup(Per User settings)

    Note
    What I did for to package Windows Live Messenger for GPO deployment (target win7 sp1)

    Download the full installer - wlsetup-all.exe
    Use resource hacker to open up wlsetup-all.exe

    In the left hand tree browse to PACKAGE
    Right click PACKAGE, save PACKAGE resources
    Save to a new temp folder
    Eg. D:\temp\package.rc

    This will output a whole lot of .bin files
    These are just cab files so we need to do a mass rename
    Ã¢Â€Âœren *.bin *.cabÃ¢Â€Â

    Once done select all cab files and extract to a new sub folder \extracted
    In \extracted you will see all the msi, msp and 7z files you need

    Files needed for WLM2011

    Requirements
    D3dx10-x86.msi
    Crt90_amd64.msi
    Crt90.msi

    WLM Application Requirements
    Contacts.msi
    WLXSuiteLang.msi
    Wllogin_wlx-x64.msi
    Wlxsuite.msi
    Pimt.msi
    Soxe.core.msi
    Soxe.definitions.msi
    Uxplatform.msi
    UxplatformLang.msi

    WLMessenger
    Messenger.msi

    Modifications
    Messenger.msi is missing its installation tables so adding via GPO doesn't work without some modifications.
    Open up Messenger.msi in Orca
    Create a New transform

    Open up another instance of Orca
    Open any working msi - in this case I used 7zip as a reference
    Export the following tables from 7zip
    ActionText
    AdminExecuteSequence
    AdvExecuteSequence
    Import these back into Messenger.msi
    Go to the Property table
    Accept the EULA

    Generate the transform .mst

    I created 3 separate GPO packages
    Software - Win 7 - Windows Live Essentials Prerequisites
    Software - Win 7 - Windows Live Essentials
    Software - Win 7 - Windows Live Essentials Messenger (make sure you apply the mst created above)

    Also ensure Software - Win 7 - Windows Live Essentials Messenger is installed last
